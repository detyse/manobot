# Manobot 操作指南

Manobot 是一个基于 nanobot 的多智能体管理系统，支持运行多个 AI 智能体并将消息路由到特定的智能体。

## 快速开始

### 1. 初始化环境

```bash
manobot init
```

首次运行会自动创建 `~/.manobot` 目录并生成默认智能体。

### 2. 查看智能体列表

```bash
manobot agents list
```

### 3. 添加新智能体

```bash
manobot agents add coder --name "代码助手"
```

### 4. 启动网关

```bash
manobot gateway
```

## CLI 命令参考

### 核心命令

| 命令 | 说明 |
|------|------|
| `manobot init` | 初始化 manobot 环境 |
| `manobot gateway` | 启动多智能体网关服务 |
| `manobot status` | 显示已配置的智能体数量和默认智能体 |
| `manobot version` | 显示版本信息 |
| `manobot agent` | 交互式对话模式 |

### Gateway 命令选项

```bash
manobot gateway [选项]

选项:
  --port, -p    网关端口 (默认: 18790)
  --verbose, -v 启用调试输出
  --agent, -a   单智能体模式，只运行指定智能体
```

**示例：**

```bash
# 使用默认设置启动
manobot gateway

# 自定义端口
manobot gateway --port 8080

# 启用调试日志
manobot gateway --verbose

# 只运行 coder 智能体
manobot gateway --agent coder
```

### 交互式对话

```bash
manobot agent [选项]

选项:
  --agent, -a   指定智能体 ID
  --session, -s 指定会话 ID
  --message, -m 单条消息模式（发送后退出）
```

**示例：**

```bash
# 与默认智能体交互
manobot agent

# 与指定智能体交互
manobot agent --agent coder

# 发送单条消息
manobot agent --message "你好"
```

## 智能体管理

### 列出所有智能体

```bash
manobot agents list

# 输出 JSON 格式
manobot agents list --json
```

### 查看智能体详情

```bash
manobot agents show <agent_id>

# 示例
manobot agents show assistant
manobot agents show coder --json
```

显示的信息包括：
- ID、名称、模型、提供商
- 工作区路径、记忆路径、会话路径
- 身份配置、技能限制

### 创建新智能体

```bash
manobot agents add <agent_id> [选项]

选项:
  --name, -n      显示名称
  --workspace, -w 工作区目录
  --model, -m     使用的 LLM 模型
  --default, -d   设为默认智能体
```

**示例：**

```bash
# 基础创建
manobot agents add coder --name "代码助手"

# 完整配置
manobot agents add writer \
  --name "写作助手" \
  --model "openai/gpt-4o" \
  --workspace ~/writing

# 创建并设为默认
manobot agents add assistant-v2 --default
```

> **提示：** 创建后需要重启网关才能生效。

### 删除智能体

```bash
manobot agents delete <agent_id>

# 跳过确认
manobot agents delete old-agent --force
```

> **注意：** 只删除配置，不会删除工作区、记忆和会话文件。

### 设置默认智能体

```bash
manobot agents set-default <agent_id>

# 示例
manobot agents set-default coder
```

默认智能体会接收所有不匹配绑定规则的消息。

## 消息路由与绑定

绑定（Bindings）用于将特定通道/聊天的消息路由到指定的智能体。

### 查看所有绑定

```bash
manobot agents bindings

# JSON 格式
manobot agents bindings --json
```

### 添加绑定规则

```bash
manobot agents bind <agent_id> [选项]

选项:
  --channel, -c   通道名称 (必须)
  --peer-type, -t 聊天类型: direct/group/channel
  --peer-id, -p   聊天 ID
  --comment       备注说明
  --force, -f     跳过验证
```

**示例：**

```bash
# 将所有 Telegram 消息路由到 assistant
manobot agents bind assistant --channel telegram

# 将特定 Telegram 群组路由到 coder
manobot agents bind coder \
  --channel telegram \
  --peer-type group \
  --peer-id -100123456789

# Discord 私聊路由到 support
manobot agents bind support \
  --channel discord \
  --peer-type direct \
  --comment "支持请求"
```

### 路由优先级

消息路由遵循以下优先级（从高到低）：

1. **EXACT_PEER** - 完全匹配 channel + peer_id
2. **PARENT_PEER** - 匹配 channel + parent_peer_id（线程/回复）
3. **GUILD_TEAM** - 匹配 channel + guild_id/team_id
4. **ACCOUNT** - 匹配 channel + account_id
5. **CHANNEL** - 仅匹配 channel
6. **FALLBACK** - 默认智能体

## 通道配置

### 查看通道状态

```bash
manobot channels status
```

显示所有支持的通道及其配置状态。

### 支持的通道

| 钉钉 | `clientId`, `clientSecret` |
| QQ | `appId`, `appSecret` |
| Email | `imapHost`, `smtpHost` |
| Matrix | `homeserver`, `accessToken` |

```bash

```

启动 WhatsApp 桥接服务，扫描二维码完成登录。

## 配置文件参考

配置文件位于 `~/.manobot/config.json`，使用 JSON 格式（支持 camelCase）。

### 完整配置示例

```json
{
  "agents": {
    "defaults": {
      "workspace": "~/.manobot/workspace",
      "model": "anthropic/claude-sonnet-4-20250514",
      "provider": "auto",
      "maxTokens": 8192,
      "temperature": 0.1,
      "maxToolIterations": 40,
      "memoryWindow": 100
    },
    "list": [
      {
        "id": "assistant",
        "default": true,
        "name": "主助手",
        "identity": {
          "name": "Nano",
          "description": "通用 AI 助手"
        }
      },
      {
        "id": "coder",
        "name": "代码助手",
        "workspace": "~/projects",
        "model": "deepseek/deepseek-coder",
        "skills": ["coding", "git", "shell"],
        "identity": {
          "name": "CodeBot",
          "description": "专业代码助手"
        }
      }
    ],
    "bindings": [
      {
        "agentId": "coder",
        "comment": "Telegram 开发群组",
        "match": {
          "channel": "telegram",
          "peerType": "group",
          "peerId": "-100123456789"
        }
      }
    ]
  },
  "channels": {
    "sendProgress": true,
    "sendToolHints": false,
    "telegram": {
      "enabled": true,
      "token": "YOUR_BOT_TOKEN"
    },
    "discord": {
      "enabled": false,
      "token": "YOUR_DISCORD_TOKEN"
    }
  },
  "providers": {
    "anthropic": {
      "apiKey": "YOUR_API_KEY"
    },
    "deepseek": {
      "apiKey": "YOUR_API_KEY"
    }
  },
  "gateway": {
    "host": "0.0.0.0",
    "port": 18790,
    "heartbeat": {
      "enabled": true,
      "intervalS": 1800
    }
  }
}
```

### 智能体配置项说明

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `id` | 唯一标识符 | (必须) |
| `default` | 是否为默认智能体 | `false` |
| `name` | 显示名称 | - |
| `workspace` | 工作区目录 | 继承默认值 |
| `model` | LLM 模型 | 继承默认值 |
| `provider` | 提供商 | `auto` |
| `maxTokens` | 最大 token 数 | `8192` |
| `temperature` | 温度参数 | `0.1` |
| `skills` | 允许的技能列表 | `null`（全部） |
| `identity` | 身份配置 | - |
| `subagents` | 子智能体配置 | - |

### 绑定匹配条件

| 字段 | 说明 |
|------|------|
| `channel` | 通道名称 (必须) |
| `accountId` | 账户 ID |
| `peerType` | 聊天类型: `direct`/`group`/`channel` |
| `peerId` | 聊天/对话 ID |
| `parentPeerId` | 父级对话 ID（用于线程） |
| `guildId` | Discord 服务器 ID |
| `teamId` | Slack 团队 ID |

## 常见操作流程

### 场景 1：多智能体分工

```bash
# 1. 创建专业智能体
manobot agents add coder --name "代码助手" --model "deepseek/deepseek-coder"
manobot agents add writer --name "写作助手" --model "openai/gpt-4o"

# 2. 设置路由规则
manobot agents bind coder --channel telegram --peer-id -100111111
manobot agents bind writer --channel telegram --peer-id -100222222

# 3. 启动网关
manobot gateway
```

### 场景 2：Discord 服务器专属智能体

```bash
# 为特定 Discord 服务器绑定智能体
manobot agents bind support \
  --channel discord \
  --comment "客服服务器"

# 配置文件中添加 guildId 匹配
# 编辑 ~/.manobot/config.json
```

### 场景 3：调试模式

```bash
# 启用详细日志
manobot gateway --verbose

# 只运行单个智能体进行测试
manobot gateway --agent coder
```

## 故障排除

### 智能体未响应

1. 检查智能体是否存在：`manobot agents list`
2. 检查绑定规则：`manobot agents bindings`
3. 启用详细日志：`manobot gateway --verbose`

### 配置修改后未生效

修改配置文件或使用 CLI 命令后，需要重启网关：

```bash
# 停止当前网关 (Ctrl+C)
# 重新启动
manobot gateway
```

### 通道连接失败

1. 检查通道配置：`manobot channels status`
2. 验证 API 密钥/Token 是否正确
3. 检查网络连接和代理设置
